## 📂 Directory Descriptions

### `/src` - Source Code
The main source code directory containing all TypeScript files.

#### `/src/config` - Configuration
- **database.ts**: PostgreSQL database configuration and connection utilities
- **env.ts**: Centralized environment variable management using dotenv

#### `/src/middlewares` - Middleware
- **error.middleware.ts**: Global error handler and 404 not found handler
- **logger.middleware.ts**: HTTP request logging middleware

#### `/src/routes` - Routes
- **health.routes.ts**: Health check endpoint implementation
- **index.ts**: Main router that aggregates all route modules

#### `/src/app.ts` - Entry Point
Main application file that:
- Initializes Express app
- Configures middleware
- Sets up routes
- Starts the HTTP server

### `/dist` - Build Output
Compiled JavaScript files generated by TypeScript compiler. This directory:
- Mirrors the structure of `/src`
- Contains `.js` files (compiled code)
- Contains `.d.ts` files (type definitions)
- Contains `.map` files (source maps for debugging)

### `/init-db` - Database Scripts
SQL scripts that run when PostgreSQL container starts:
- **01-init.sql**: Creates initial database structure and test data

### Root Files

#### Configuration Files
- **tsconfig.json**: TypeScript compiler options and settings
- **package.json**: Node.js project metadata and dependencies
- **.env**: Environment variables (local, not committed to git)
- **.env.example**: Template for environment variables

#### Docker Files
- **Dockerfile**: Multi-stage Docker build configuration
- **docker-compose.yml**: Orchestrates multiple containers (API, DB, pgAdmin)
- **.dockerignore**: Files to exclude from Docker build context

#### Documentation
- **README.md**: Comprehensive project documentation
- **QUICKSTART.md**: Quick start guide for developers
- **STEP1-COMPLETION.md**: Summary of Step 1 implementation
- **PROJECT-STRUCTURE.md**: This file - detailed structure explanation

#### Other
- **.gitignore**: Files to exclude from version control
- **package-lock.json**: Locked versions of dependencies

## 📝 File Naming Conventions

- **TypeScript files**: `kebab-case.ts` (e.g., `error.middleware.ts`)
- **Route files**: `resource.routes.ts` (e.g., `health.routes.ts`)
- **Middleware files**: `name.middleware.ts` (e.g., `logger.middleware.ts`)
- **Config files**: `name.ts` (e.g., `database.ts`)
- **Type files**: `name.types.ts` (e.g., `user.types.ts`)

## 🎯 Architecture Pattern

The project follows a **layered architecture**:

```
┌─────────────────────────────────────┐
│         Routes (API Layer)          │  ← HTTP endpoints
├─────────────────────────────────────┤
│      Controllers (Logic Layer)      │  ← Request handling
├─────────────────────────────────────┤
│      Services (Business Layer)      │  ← Business logic
├─────────────────────────────────────┤
│       Models (Data Layer)           │  ← Database operations
└─────────────────────────────────────┘
```

## 🔐 Security Considerations

- `.env` files are excluded from version control
- Sensitive data is stored in environment variables
- Docker containers run as non-root users
- Security headers are applied via Helmet middleware

## 🚀 Build Process

1. **Development**: `npm run dev`
   - Uses `tsx` to run TypeScript directly
   - Hot reload on file changes
   - No compilation step

2. **Production**: `npm run build` → `npm start`
   - TypeScript compiles to `/dist`
   - Runs compiled JavaScript
   - Optimized for production
